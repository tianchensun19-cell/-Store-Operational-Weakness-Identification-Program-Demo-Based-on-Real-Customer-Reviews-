# ReviewPulse CN Pro Demo（评委说明版）

本项目是一个面向**门店运营者**的中文评论风险识别 Demo，用于演示如何通过顾客评论的语义聚类、基线对比和期对期对比，快速识别当前门店最需要优先处理的运营问题。[web:397][web:459]

## 假设情景

本 Demo 基于以下业务场景设计：

- 一家连锁餐饮店最近在新片区中开了一个新门店，他们不确定该区域的外卖配送压力和顾客口味偏好。
- 为了更好地盈利，新门店管理者每周会对顾客评论进行分析，识别当前运营模式是否存在不足，并及时改进。
- 总店提供了 3000 条顾客评论，作为门店运营的基线情况。
- 通过将每周顾客评论与基线情况和上周情况对比，可以获得顾客关注点的变化趋势和最亟需改进的方面。

## Demo 能做什么

本系统会自动完成以下工作：

- 对评论内容进行语义聚类，识别评论集中反映的问题类别。
- 将当前期评论与固定 baseline 对比，判断哪些问题高于基线水平。
- 将当前期评论与上一期结果对比，识别哪些问题正在恶化或改善。
- 自动识别新出现的问题类别，并输出优先级和整改建议。
- 对小样本问题做平滑处理，减少单条极端评论带来的误判。

## 数据说明

本 Demo 使用三类数据：

| 数据类型 | 作用 | 说明 |
|---|---|---|
| `baseline.xlsx` | 固定基线 | 用于代表总店历史运营基线，不会在演示过程中自动更新。 |
| `sample_data/2026_period_1_demo.xlsx` | 第 1 期样例 | 用于演示首期分析结果。 |
| `sample_data/2026_period_2_demo.xlsx` | 第 2 期样例 | 用于演示与上期、与 baseline 的双对比。 |
| `sample_data/2026_period_3_demo.xlsx` | 第 3 期样例 | 用于演示问题演化与潜在新问题识别。 |

## 建议演示方式

打开应用后，可通过以下三种方式体验：

### 方式一：一键演示

页面中为第 1 期、第 2 期、第 3 期分别提供“**一键载入**”按钮。评委点击后即可直接进入对应期数的分析结果，无需准备本地文件。

### 方式二：下载并上传样例数据

页面中为每一期分别提供“**下载样例**”按钮。评委可以先下载 Excel，再通过“**手动上传**”体验完整的数据上传与分析流程。

### 方式三：上传自定义数据

若希望进一步测试，也可以上传与样例结构一致的评论数据文件。建议至少包含以下字段：

- `评价内容`
- `评价类型`

## 推荐体验顺序

为了更清楚地理解系统价值，建议按以下顺序体验：

1. 点击“第 1 期一键载入”，观察系统如何识别当前主要问题。
2. 点击“第 2 期一键载入”，观察系统如何展示相对 baseline 和相对上期的变化。
3. 点击“第 3 期一键载入”，观察系统如何识别问题演化趋势和新问题。

这种顺序可以更完整地体现本系统在**周度运营复盘**中的使用方式。

## 评估重点

建议从以下角度观察本 Demo：

- 是否能快速指出当前最严重问题。
- 是否能区分“长期高风险问题”和“本周突然恶化问题”。
- 是否能帮助门店管理者理解顾客关注点的变化趋势。
- 是否能将评论分析结果转化为可执行的整改建议。

## 技术实现要点

本 Demo 基于 Streamlit 构建，并部署为可直接访问的网页应用；官方支持通过 GitHub 仓库将 Streamlit 应用部署到 Community Cloud，并生成可分享链接。[web:397][web:398][web:459]

在实现上，系统重点采用了以下策略：

- 中文评论语义向量编码；
- 聚类后问题命名与去重；
- 当前期对 baseline 的语义匹配；
- 当前期对上一期的风险变化比较；
- 小样本平滑与优先级排序。

## 使用提示

- 首次打开应用时，会先显示产品说明弹窗。
- 关闭说明后，系统会初始化 baseline，并进入正式分析页面。
- 若样例数据文件名称与预设路径一致，页面中的“下载”“一键载入”“手动上传”功能都可直接使用。
